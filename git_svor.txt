Git spurningar og svör
1. Hvað gera eftirfarandi Linux skipanir?
	cd breytir directory
	cd ~ fer tilbaka um directory
	ls byrtir lista
	pwd sýnir hvaða directory er virkt
	mkdir býr til directory

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone hleður niður verkefni og öllum útgáfum þess.
	git log sýnir útgáfur verkefnis
	git status byrtir allar nýjar skrár eða þær sem hafa verið breytt sem eiga eftir að vera "commit-ed"
	git diff sýnir mismun skráar sem hafa ekki en verið sviðsettar
	git checkout Fer í branch og uppfærir vinnandi directory

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff Shows file differences not yet staged
	b) git diff --staged Shows file differences between staging and the last file version
	c) git diff commit1 commit2 Shows file differences not yet staged only from specified commits

4. 	Hvað er útgáfustýring (e.version control)? 
		kerfi sem loggar breytingar á skrám svo hægt er að kalla þær upp aftur

5.	Hverjir eru helstu kostir við að nota GIT?
		auðveldar tveim+ einstaklingum að vinna í sama verkefni þó þeir séu kanski hinumeigin á hnettinum

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
		þegar notandi er sáttur með færslu, því ef eitthvað skildi fara úrskeiðis er hægt að fara tilbaka

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
		working dir. er þegar er verið að vinna. staging area er áður en það er commit-að fyrir fullt og allt, og repository er þegar það er búið að commit-a.

8.	Hvenær er sniðugt að nota greinar (branches)? 
		þegar það er t.d. verið að vinna í verkefni en notandi vill ekki over-write-a aðal verkefnið.
